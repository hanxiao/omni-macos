# Omni - project notes

Native macOS app for semantic search over local files, powered by an in-process
MLX-Swift port of `jinaai/jina-embeddings-v5-omni-small-mlx`.

## Hard constraints
- Native macOS SwiftUI app, native macOS UI idioms. No Python at runtime.
- Embeddings run in-process via MLX-Swift (mlx-swift), not via any server.
- Indexes user files (Documents/Downloads/Desktop, user-selectable).
- Text first; for non-extractable PDFs (scans) render page -> image -> omni image embedding.

## Architecture
- `OmniKit` (SPM library): the engine + indexer.
  - `OmniTextEncoder`: Qwen3 text tower, retrieval LoRA merged at load, last-token pool + L2.
  - `OmniImageEncoder`: Qwen3-VL ViT + merger, feature injection (for scanned PDFs).
  - `WeightStore`: loads HF safetensors, merges retrieval LoRA, upcasts backbone to fp32.
  - Indexer: crawler + text extraction + SQLite vector store + cosine search.
- `Omni` (Xcode app target, generated by XcodeGen from `project.yml`).
- `omni-verify` (SPM executable): numeric validation against Python fixtures.

## Verification (measured, never assumed)
- Fixtures generated by `Tools/gen_fixtures.py` from the reference model (run in the mls venv).
- Text: MLX-Swift must hit cosine >= 0.999 vs reference; token ids must match exactly.
- Image: MLX-Swift must match Python `model.py` on a test image.
- Run: `swift run omni-verify <modelDir> Fixtures/text_fixtures.json`

## Model layout (reference, on this machine)
- Snapshot: `/Volumes/One Touch/ai-models/huggingface/hub/models--jinaai--jina-embeddings-v5-omni-small-mlx/snapshots/<rev>`
- Needs: `model.safetensors`, `adapters/retrieval/adapter_model.safetensors`, `tokenizer.json`, `config.json`.
- Retrieval LoRA: alpha=32, r=32 -> scale 1.0, targets all 7 linear modules in `language_model`.

## Signing (no Apple creds in the repo - it is public)
- Local: `export OMNI_TEAM_ID=<10-char team>` before `xcodegen generate` (project.yml reads it).
- CI: Team ID is the `APPLE_TEAM_ID` secret; cert/notary/deploy creds are GitHub Actions secrets
  (DEVELOPER_ID_P12, DEVELOPER_ID_P12_PASSWORD, AC_APPLE_ID, AC_PASSWORD, SITE_TOKEN).
- notarize.sh takes AC_* creds from env (no keychain profile) so the self-hosted runner works.
